## Twelve Sample Checks

1. Record the exact build, URL, and critical user flow before testing.
2. Confirm success ends in an unambiguous state.
3. Verify repeated submission does not create unintended duplicates.
4. Check recoverable validation errors preserve user-entered content.
5. Test for unintended horizontal scrolling at 320 px width.
6. Confirm sticky UI does not hide the active input or primary action.
7. Reach every control with a keyboard and verify visible focus.
8. Check icon-only buttons have an accessible name.
9. Verify expandable controls expose expanded/collapsed state.
10. Tie new console or network failures to visible user impact.
11. Check every in-page hash link has a real target.
12. Record untested areas and environment limitations in the final handoff.
